# 自分用の音楽共有サイト

## やりたいこと

- どこでも好きな音楽を聞けるように、自分用に音楽共有サイトをサーバレスに作りたい
- 維持費は可能限りゼロに近づける
- 共有したい曲は `music-library/` に保存
  - **アーティストごと** にディレクトリを分けている
    - アーティストの並び順は **昇順** で表示する。
    - フォルダ名でナンバリングしていますが、はじめの数字とアンダースコア（ `99_` ）は管理用のナンバリングなのでサイトには表示したくない
  - さらにサブディレクトリで **アルバム** を分けている
  - アルバム内の曲の順番は **昇順** でソートして聞きたい
- アクセス可能なユーザを制限するため、簡単なパスワードを設定しておく。指定したいパスワードは `.env` ファイルで設定する。

## サイトのデザイン

- サイドバーに曲の一覧が表示されている
- 右側のコンテンツウィンドウで、曲のサムネイルと再生のコントロールができるようにする（これは YouTube Music のデザインを真似する）
- パソコンのブラウザでも、スマホのブラウザでも、快適に操作できるようにデザインを考える
- アートワーク（サムネイル）は、音楽ファイルに含まれている場合は、そのまま使います。
- アートワーク（サムネイル）が無い曲は、 **Nano banana Pro** で画像を生成します。
  - API Key は Google AI Studio で発行した Key を使用します。API Key は **.env** に書いています。
  - 楽曲の情報が必要な場合は **askuserquestion** でユーザに確認します。

## 機能面の希望

- 再生を始めたら、直ぐに再生できるようにしたい
- アーティストごとに、 **アルバムと曲名の昇順で、連続再生できる** ようにしたい
- YouTube Music のように **いいね** や、 **お気に入り** 登録ができるようにしたい
- **最初からおすすめ** で表示するのは次の曲（またはアルバム）
  - `music-library/01_名取交響吹奏楽団（n響）` の全ての曲を連続再生
  - `music-library/02_シエナ・ウィンド・オーケストラ/バーンスタイン・オン・ブラス/11 プレリュード、フーガ&リフス　プレリュード.m4a` から連続再生
  - `music-library/03_東北方面陸上自衛隊/第31回青少年コンサート/07 クラリネット協奏曲　Carousel.mp3` から連続再生
  - `music-library/04_航空自衛隊西部航空音楽隊/CAFUA Selection 2004 吹奏楽コンクール自由曲選「ジェリコ」/05 スクーティン・オン・ハードロック.mp3` から連続再生
- 別環境でも聞けるように、 **楽曲をダウンロードできる** ようにしたい
